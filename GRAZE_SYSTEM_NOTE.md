# Graze System Note

> Status: obsolete / archived

---

## Purpose

This file is kept only as archived design context.

Graze has been removed from the current game direction.

It is no longer an active mechanic the team should design around or implement against.

---

## Why This File Still Exists

This note remains on disk so future agents understand:
- graze used to exist
- graze previously fed OverDrive
- the design moved away from it after repeated playtesting

The problem was not only implementation cost.
The bigger problem was that the mechanic did not become meaningfully central to real play.

In practice, the player could ignore it for long stretches or entire runs.

That made it a weak fit for the final direction.

---

## Current Direction Instead

The game has shifted toward:
- dash-based heat rhythm
- kill-driven OverDrive buildup
- clearer movement-based flow
- stronger visible payoff states

So the old graze role has effectively been replaced by:
- dash preserving flow
- combat momentum building OverDrive

---

## Instruction For Future Agents

Do not:
- reintroduce graze assumptions into new docs
- make OverDrive depend on graze
- treat this file as an active spec

If graze is ever reconsidered in the future, it should be treated as a fresh design problem, not revived automatically from this file.

---

## Summary

Graze is no longer part of the active design direction.

This file exists only so agents understand the history of that decision.
